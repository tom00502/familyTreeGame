## 互動元件設計規範

### 數量輸入 — Stepper（加減按鈕）

所有需要使用者輸入「數量」的場景，**一律使用加減按鈕（Stepper）**，禁止使用裸 `<input type="number">` 或自由文字輸入框。

#### 設計結構

```
[ − ]  數字  [ + ]
```

- **減號按鈕**：圓形，`w-10~12 h-10~12 rounded-full`，border `border-[#8B8278]/20`
- **數字顯示**：置中，大字號粗體 `text-3xl~4xl font-bold text-[#8B2635]`，最小寬度 `min-w-[2.5rem~3rem]`
- **加號按鈕**：與減號同尺寸與樣式
- **按鈕互動**：hover 時淺色背景 `hover:bg-[#F5F1E8]`；減號在最小值時 `disabled:opacity-30`

#### 行為規則

| 規則   | 說明                                                         |
| ------ | ------------------------------------------------------------ |
| 最小值 | 視情境而定（通常 `0` 或 `1`），到達最小值時減號按鈕 disabled |
| 最大值 | 設上限防止誤觸（建議 `20`）                                  |
| 初始值 | 預設 `0`；若情境暗示「至少 1」，則首次點擊自動設為 `1`       |
| 無鍵盤 | 不需要鍵盤輸入，純按鈕操作，觸控友善                         |

#### 適用場景

- Phase 2 — 子女數量詢問（downward-inquiry）
- Phase 3 — 子女數量驗證（children-count）
- Phase 3 — 兄弟姐妹數量驗證（sibling-verify）
- 任何未來新增的「回答一個整數」題型

#### 設計理由

1. **觸控友善**：大圓形按鈕比小型 number input 更容易在手機上操作
2. **視覺一致**：統一的 Stepper 樣式讓不同階段的數字題感覺一致
3. **防誤輸入**：避免使用者輸入非數字、負數或不合理的巨大數字
4. **即時反饋**：每次按壓數字立刻變化，體驗直覺
