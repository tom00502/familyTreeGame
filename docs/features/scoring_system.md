# 計分系統 (Scoring System)

> ⚠️ **本文件為計分系統的唯一權威定義**。其他文件（Phase 2、Phase 3）不重複定義計分規則，僅引用本文件。

## 📌 概述

計分系統評估每位玩家在遊戲中的**貢獻度**，並於「大揭曉」階段呈現個人排名與 MVP。

分數來源分為四層：

1. **任務基礎分**：完成任務即得分，依任務類型而異
2. **獎勵加分**：速度、首次填寫、貢獻度等額外獎勵
3. **準確性修飾符**：根據答案與最終結果的一致性進行加成或減成
4. **扣分機制**：跳過、超時、衝突駁回的懲罰

> 📎 **名詞說明**：本文件的「分數」指玩家的**最終積分 (Final Score)**，用於排名與 MVP 評定。Phase 2 任務派發中的「任務配對優先分」為另一套獨立的系統內部分數，與玩家積分無直接關係，詳見 [qa_phase2_data_filling.md](./qa_phase2_data_filling.md)。

## 📊 任務基礎分

各任務類型的基礎分依**對族譜的貢獻程度**遞增排列：

| 任務類型                     | 基礎分 | 說明                                   |
| ---------------------------- | ------ | -------------------------------------- |
| 排序確認 (Age Ordering)      | 0.5 分 | 輔助任務，對結構影響最小               |
| 節點命名 (Node Naming)       | 1 分   | 虛擬節點命名，基本資料填充             |
| 屬性填充 (Attribute Filling) | 1 分   | 補齊性別、生日等屬性                   |
| 回答關係問題 (Phase 1)       | 1 分   | 每個正確回答的關係確認問題             |
| 向上追溯 (Upward Tracing)    | 1.5 分 | 路徑閉合，連結上一代節點               |
| 節點匯聚 (Node Convergence)  | 2 分   | 確認多路徑匯聚點，對族譜正確性影響最大 |

## 🏆 獎勵機制

### 速度獎勵

| 提交時間       | 加分  |
| -------------- | ----- |
| 10 秒內提交    | +5 分 |
| 20 秒內提交    | +3 分 |
| 超過 20 秒提交 | 無    |

### 首次填寫獎勵

同一個節點的同一項資訊，**首位填寫者**獲得完整基礎分，後續填寫者僅獲得確認獎：

| 填寫順序 | 得分              | 說明           |
| -------- | ----------------- | -------------- |
| 首次填寫 | 100% × 任務基礎分 | 完整積分       |
| 確認填寫 | 5% × 任務基礎分   | 確認獎（微量） |

```
例：「爺爺的生日」（屬性填充，基礎分 = 1 分）
  - 玩家 A（首次填寫）：獲得 1 × 100% = +1 分
  - 玩家 B（確認填寫）：獲得 1 × 5% = +0.05 分
  - 玩家 C（確認填寫）：獲得 1 × 5% = +0.05 分
```

### 貢獻度獎勵

- 完成所有派給自己的任務（一次跳過都沒有）：遊戲結束後 **+10 分**

## 📐 準確性修飾符

遊戲結束時（Phase 3），系統根據**衝突解決結果**對每筆答案進行加成或減成。

### 多數決加成

若填寫的資訊與最終採用的結果**一致**，獲得加成獎勵：

| 情況                         | 修飾符 | 說明     |
| ---------------------------- | ------ | -------- |
| 答案與最終結果一致（多數）   | × 1.2  | 加成 20% |
| 答案與最終結果不一致（少數） | × 0.5  | 減成 50% |

```
例：「奶奶的生日」（屬性填充，基礎分 = 1 分）
  - 填寫 1950-01-01 的玩家（多數）：獲得 1 × 1.2 = +1.2 分
  - 填寫 1950-01-02 的玩家（少數）：獲得 1 × 0.5 = +0.5 分
```

### 衝突駁回扣分

若填寫的資訊在衝突解決中被**駁回**（最終未採用），扣除原應獲得積分的 50%：

```
例：「叔叔的性別」有衝突（玩家 A 說男性，玩家 B 說女性）
  - 最終採用玩家 A 的「男性」：
    玩家 A 獲得 +1 分（無扣分）
    玩家 B 扣除 -0.5 分（原應得 1 分的 50%）
```

---

## ⬇️ 扣分機制

### 跳過懲罰

- 每次主動跳過任務：**-2 分**
- 玩家可以無限次跳過，**無冷卻限制**
- 跳過後任務重新進入派發佇列，優先級降低（詳見 [qa_phase2_data_filling.md](./qa_phase2_data_filling.md) 的任務派發策略）

### 超時懲罰

- 玩家獲派任務後 30 秒無任何動作，系統自動作為跳過處理：**-2 分**

## 🧮 最終分數計算

```
Final Score = Σ(任務基礎分 × 首次/確認係數 × 準確性修飾符) + 速度獎勵 + 貢獻度獎勵 - 跳過懲罰 - 衝突駁回扣分
```

### 積分計算示例

```
玩家 A 的積分計算：

活動記錄：
1. 第一階段：回答「玩家 B 是我的堂兄」→ 1 × 100% = +1 分
2. 第二階段：填寫「爺爺的名字」（首次）→ 1 × 100% = +1 分
3. 第二階段：填寫「爺爺的生日」（確認）→ 1 × 5% = +0.05 分
4. 第二階段：向上追溯「祖父的父親是誰」→ 1.5 × 100% = +1.5 分
5. 第二階段：確認「叔叔是王金剛」（節點匯聚）→ 2 × 100% = +2 分
6. 第二階段：填寫「爺爺的生日」（多數決加成）→ 1 × 1.2 = +1.2 分
7. 第二階段：排序「爺爺 vs 奶奶年齡」→ 0.5 × 100% = +0.5 分

總積分：1 + 1 + 0.05 + 1.5 + 2 + 1.2 + 0.5 = 7.25 分
```

## 🥇 排名與 MVP

### 排名決定

```
1. 按「最終分數 (Final Score)」從高到低排序
2. 積分相同時，按「任務完成數」（填寫答案數）排序
3. 若仍相同，按「平均積分/任務」排序（效率排序）
```

### 特殊獎項

**MVP（Most Valuable Player）**：最終分數最高的玩家

```
獎勵：
  - 特殊標誌：👑 或 ⭐
  - 額外榮譽：「最佳貢獻者」
  - 可選遊戲獎勵：解鎖成就、積分加 10% 等
```

**效率王**：平均每個任務擁有最高積分（效率最高）

```
計算：
  平均積分 = 最終分數 / 完成任務數

例：
  玩家 A：7.25 分 / 7 任 ≈ 1.04 分/任
  玩家 B：8.5 分 / 12 任 ≈ 0.71 分/任
  → 效率王是玩家 A
```

**團隊英雄**：協作獎（任務分散度最高，幫助多個領域）

```
計算：
  - 參與任務類型的多樣性
  - 幫助他人（通過確認填寫）的次數

若某玩家在所有任務類型都有貢獻，即為團隊英雄
```
