# 階段 4：結果揭曉 (Big Reveal)

## 📌 階段概述

**觸發時機**：遊戲設定的時間（90/120/180 秒）耗盡，Phase 3（資料驗證）結束

**玩家體驗**：遊戲暫停所有提問，畫面逐漸轉換為揭曉模式。系統以動畫展開完整的家族族譜，最後顯示排行榜與個人貢獻統計

**系統目標**：

1. 以視覺化動畫揭曉完整族譜結構
2. 彙總 Phase 1-3 的所有計分，產出最終排名
3. 展示個人貢獻、族譜完整度與特殊獎項
4. 提供族譜的分享、匯出與保存功能

---

## 🎬 揭曉動畫

### 動畫序列

#### 階段 4.1：畫面轉換（0-1 秒）

```
遊戲時間耗盡，立即暫停所有提問
UI 逐漸淡出（fade out）所有問題卡片
背景漸變為「族譜展示」模式（顏色變淡、聚焦中央）
```

#### 階段 4.2：登場動畫（1-3 秒）

```
從現場玩家節點開始，逐漸「點亮」：
  - 第 0 層：所有玩家節點同時點亮（綠色光暈，伸縮 1.5 倍）
  - 第 1 層：父母節點逐個點亮（0.5 秒間隔），伴隨連線動畫
  - 第 2 層：祖父母與配偶節點逐個點亮
  - 第 3 層：更上層與旁系親屬逐個點亮

視覺效果：
  - 每個新點亮的節點伴隨「漣漪」動畫（向外擴散的光波）
  - 連線從父到子逐條繪製（路徑動畫，線條漸變而不是突現）
  - 背景音：溫暖的弦樂及家族合奏聲（可選）
```

#### 階段 4.3：完整族譜展示（3-5 秒）

```
族譜完全展開，呈現以下資訊：
  - 所有節點位置最終定位（已完成布局演算法）
  - 每個節點顯示：姓名、性別圖示、出生年份（如有）
  - 連線以不同顏色區分：
    * 紅線：血緣關係（父子）
    * 橙線：配偶關係
    * 虛線：待確認的關係
    * 灰線：有衝突標記的關係

玩家視角：
  - 整個家族樹從無到有地「生長」在眼前
  - 可以看到自己在族譜中的位置及與他人的關係
  - 驚喜與感動（家族連接的視覺化）
```

#### 階段 4.4：過渡到結果頁面（5-6 秒）

```
族譜保持在螢幕中央，逐漸收縮至左側（留出空間顯示結果）
右側漸漸顯示計分板與排名（從透明逐漸顯示）
完整的結果頁面呈現
```

## 📊 計分與排名

> 📄 **詳細計分規則** → 請查閱 [scoring_system.md](./scoring_system.md)

Phase 4 時，系統自動彙總 Phase 1-3 的所有計分，計算每位玩家的**最終分數 (Final Score)**，並據此排名。計分過程包括：

1. 累計 Phase 1 的關係確認得分
2. 累計 Phase 2 各任務的基礎分（依任務類型而異）
3. 累計 Phase 3 的驗證得分（驗證正確、發現錯誤獎勵等）
4. 應用首次填寫係數、準確性修飾符
5. 加入速度獎勵與貢獻度獎勵
6. 扣除跳過、超時與一致性懲罰
7. 依最終分數排序，授予 MVP、效率王、團隊英雄等特殊獎項

## 🎯 特殊情況與異常處理

### 情況 1：多族譜分支（不連通）

**現象**：若所有玩家之間都互相「完全不知道」，可能產生多個獨立的族譜分支

**顯示方式**：

```
族譜頁面顯示多棵樹：
  [分支 1：玩家 A、B、C 的族譜樹]
  [分支 2：玩家 D、E 的族譜樹]

說明文字：「遊戲發現 2 個獨立的家族分支」
```

**計分影響**：無影響，按正常計分規則

### 情況 2：所有玩家都是同輩（無法確認上代）

**現象**：玩家都無法填寫上代資訊，族譜只有「現場玩家層」

**顯示方式**：

```
族譜顯示：
  - 中央：5 個玩家節點（同層排列）
  - 上層：虛擬節點「父親」與「母親」
  - 無更上層延伸

提示文字：「這組玩家的上代資訊無法充分確認」
```

**計分影響**：無影響，系統正常計分

### 情況 3：完整族譜無法閉合

**現象**：某些玩家無法透過虛擬節點連接到其他玩家（MVFT 未達成）

**警告顯示**：

```
[⚠️ 族譜不完整]

無法確認的關係：
  • 玩家 A 與 B 的連接路徑
  • 玩家 C 的上代資訊

原因可能：
  1. 玩家間完全不知道彼此
  2. 資訊數量不足以構建路徑
  3. 存在未解決的衝突
```

**計分影響**：

- 已填寫的部分正常計分
- 無法填寫的部分不被懲罰
- 可獲得「努力獎」：在不完整的資訊下還是盡力貢獻

### 情況 4：高衝突情況

**現象**：Phase 3 結束後仍有多個未解決的衝突

**警告顯示**：

```
[⚠️ 資料驗證警告]

檢測到 N 個未解決衝突：
  • X 個高優先級（關係衝突）
  • Y 個中優先級（屬性衝突）

族譜以「驗證後最佳版本」呈現
部分關係可能仍存在不準確
```

**顯示方式**：衝突節點用紅色邊框標記，可點擊查看詳情

**計分影響**：無直接影響，衝突標記僅作為視覺提示

### 情況 5：旁觀者

**現象**：旁觀者觀看揭曉畫面

**顯示方式**：

```
旁觀者頁面：
  - 同步觀看族譜揭曉動畫
  - 可查看完整排行榜與族譜
  - 不出現在排行榜上
```

**計分影響**：旁觀者無積分，不出現排行榜

    "scores": ["..."],
    "ranking": ["..."],
    "mvft_status": true,
    "verification_coverage": 0.67,
    "conflicts": ["..."]

### 回顧與分享

玩家可以：

- 📸 截圖或下載族譜圖片
- 📊 導出數據（CSV/JSON）進行個人保存
- 🔗 生成分享連結（可讓他人查看該遊戲的族譜，但無法修改）
- 💾 保存至個人帳號（如有帳號系統）

## 📋 遊戲結束後

### 玩家操作

✅ **查看完整族譜**：首次能看到完整的族譜結構

✅ **檢視個人貢獻**：詳細查看自己在三個階段的每項貢獻與積分

✅ **檢視衝突詳情**：點擊衝突節點，查看不同玩家的答案與系統判定

✅ **分享與導出**：分享族譜圖片或導出數據

✅ **返回大廳**：準備進行下一局遊戲

❌ **修改答案**：遊戲結束後無法修改已提交的答案

❌ **切換房間**：必須返回大廳，無法在結果頁面切換房間
